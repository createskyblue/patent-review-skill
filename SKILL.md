# 专利审核

## 审核流程

### 1. 确定专利类型
- **发明专利**：技术方案类，包括方法、算法、产品结构（需具备新颖性、创造性、实用性）
- **实用新型**：产品形状、构造改进，不需要算法（仅需新颖性、实用性）
- **外观设计**：产品外观造型、图案

### 2. 专利三性判断（核心！）

#### 新颖性
- 该发明在申请日以前**未被公开**（包括出版物公开、使用公开、以其他方式为公众所知）
- 没有同样的发明由他人向专利局提出过申请并记载在申请日以后公布的专利文件中

#### 创造性
- 同申请日以前已有的技术相比，该发明有**突出的实质性特点**和**显著的进步**
- 不是现有技术通过简单的分析、归纳、推理就能自然获得的结果

#### 实用性
- 该发明能够**制造或者使用**
- 能够产生**积极效果**（经济效益，社会效益）

### 3. 核心审核要点

#### 逻辑合理性检查
- 背景技术描述是否准确，现有技术存在的问题是否真实
- 发明目的与解决的技术问题是否对应
- 技术方案是否能够解决所述技术问题
- 实施方式是否有逻辑矛盾

#### 技术可行性检查
- 技术手段是否在现有技术基础上可以实现
- 关键技术参数是否合理
- 原理描述是否与物理/化学原理相符

#### 创新点评估
- 创新点是否清晰、具体、可实现
- 是否存在与现有技术的实质性区别
- 保护范围是否适当

### 4. 常见问题识别

| 问题类型 | 检查点 |
|---------|--------|
| 逻辑漏洞 | 前后矛盾、因果倒置、跳跃性结论 |
| 技术错误 | 违反基本原理、参数不合理、无法实现 |
| 描述不足 | 核心方案模糊、实施例太少、关键技术参数缺失 |
| 保护不当 | 范围过宽或过窄、缺乏层次 |
| 新颖性问题 | 与现有技术过于相似，缺乏创新 |
| 创造性问题 | 简单改进未达到"实质性特点"标准 |

### 5. 专利交底书必查项

根据专利审查标准，一份合格的交底书应包含：

| 必查项 | 检查内容 |
|--------|---------|
| 发明名称 | 简洁明了，反映技术内容，不超过25字 |
| 所属技术领域 | 明确技术领域，符合国际分类 |
| 背景技术 | 客观描述现有技术状况，指出问题和缺点 |
| 发明目的 | 清晰说明要解决的技术问题 |
| 技术方案 | 详细、完整，结合附图说明 |
| 技术效果 | 明确技术方案带来的积极效果 |
| 关键创新点 | 明确与现有技术的区别 |

### 6. 输出格式

请按以下结构输出审核结果：

```
## 审核结果

### 一、专利类型
[发明/实用新型/外观设计]

### 二、专利三性判断
- 新颖性：[符合/不符合] - [评估理由]
- 创造性：[符合/不符合] - [评估理由]
- 实用性：[符合/不符合] - [评估理由]

### 三、创新点摘要
[简述本专利的核心创新]

### 四、问题清单
1. [问题1描述]
   - 问题类型：[逻辑/技术/描述/保护/新颖性/创造性]
   - 严重程度：[高/中/低]
   - 建议修改：[修改建议]

2. [问题2描述]
   ...

### 五、总体评价
[优/良/需修改]
```

## 使用方式

用户提供专利文档后，直接按照上述审核流程进行审核，无需额外步骤。
